Langkah 1 – Menghitung Probabilitas Prior

Probabilitas prior dihitung berdasarkan jumlah kemunculan setiap kelas dalam 15 record sampel training.

P(MASUK)       = 7 / 15 = 0,4667
P(TIDAK MASUK) = 8 / 15 = 0,5333

Langkah 2 – Menghitung Probabilitas Kondisional untuk Kelas MASUK

Probabilitas kondisional dihitung menggunakan Laplace Smoothing dengan rumus P(xi | Ck) = (nik + 1) / (nk + K), di mana nik adalah jumlah data kelas Ck yang memiliki nilai xi, nk adalah total data kelas Ck, dan K adalah jumlah nilai unik pada atribut tersebut dalam sampel.

Berdasarkan rekap pada Tabel 4.9, perhitungan probabilitas kondisional untuk kelas MASUK adalah sebagai berikut.

P(Jarak = Dekat             | MASUK) = (2 + 1) / (7 + 3) = 3 / 10  = 0,3000
P(Follow Up = Belum Dihubungi | MASUK) = (6 + 1) / (7 + 4) = 7 / 11  = 0,6364
P(Status Test = Sudah Tes     | MASUK) = (7 + 1) / (7 + 2) = 8 / 9   = 0,8889
P(Nilai = Nilai Tinggi        | MASUK) = (6 + 1) / (7 + 3) = 7 / 10  = 0,7000
P(Penghasilan = >Rp6.000.000 | MASUK) = (3 + 1) / (7 + 3) = 4 / 10  = 0,4000

Langkah 3 – Menghitung Probabilitas Kondisional untuk Kelas TIDAK MASUK

P(Jarak = Dekat             | TIDAK MASUK) = (1 + 1) / (8 + 3) = 2 / 11 = 0,1818
P(Follow Up = Belum Dihubungi | TIDAK MASUK) = (5 + 1) / (8 + 4) = 6 / 12 = 0,5000
P(Status Test = Sudah Tes     | TIDAK MASUK) = (3 + 1) / (8 + 2) = 4 / 10 = 0,4000
P(Nilai = Nilai Tinggi        | TIDAK MASUK) = (2 + 1) / (8 + 3) = 3 / 11 = 0,2727
P(Penghasilan = >Rp6.000.000 | TIDAK MASUK) = (0 + 1) / (8 + 3) = 1 / 11 = 0,0909

Langkah 4 – Menghitung Skor Akhir Setiap Kelas

Skor akhir diperoleh dari perkalian probabilitas prior dengan seluruh probabilitas kondisional masing-masing kelas.

Kelas MASUK:
P(MASUK | X) = P(MASUK) × P(Dekat|M) × P(Belum Dihubungi|M) × P(Sudah Tes|M) × P(Nilai Tinggi|M) × P(>Rp6jt|M)
                = 0,4667 × 0,3000 × 0,6364 × 0,8889 × 0,7000 × 0,4000
                = 0,022174
Kelas TIDAK MASUK:
P(TIDAK MASUK | X) = P(TM) × P(Dekat|TM) × P(Belum Dihubungi|TM) × P(Sudah Tes|TM) × P(Nilai Tinggi|TM) × P(>Rp6jt|TM)
                      = 0,5333 × 0,1818 × 0,5000 × 0,4000 × 0,2727 × 0,0909
                      = 0,000481

Langkah 5 – Penentuan Kelas

Kelas dipilih berdasarkan nilai skor tertinggi di antara kedua kelas.

P(MASUK | X)       = 0,022174
P(TIDAK MASUK | X) = 0,000481

0,022174 > 0,000481
Hasil Klasifikasi: Karena P(MASUK | X) > P(TIDAK MASUK | X), maka calon mahasiswa tersebut diprediksi masuk kelas MASUK. Hasil prediksi ini sesuai dengan status aktual data uji yang bersangkutan.

5. Evaluasi Model

Model Naive Bayes yang dibangun dari 1.028 data training selanjutnya diuji menggunakan 257 data testing. Hasil klasifikasi model terhadap data testing disajikan dalam bentuk confusion matrix pada Tabel 4.15.

Tabel 4.15 – Confusion Matrix Hasil Klasifikasi Naive Bayes
Prediksi
MASUKTIDAK MASUK
Aktual MASUK 74 (TP) 0 (FN)
TIDAK MASUK 16 (FP) 167 (TN)
Sumber: Hasil Pengolahan Data (2025)

Berdasarkan confusion matrix pada Tabel 4.15, diperoleh nilai TP = 74, TN = 167, FP = 16, dan FN = 0. Nilai-nilai ini selanjutnya digunakan untuk menghitung metrik evaluasi model.

a. Akurasi

Akurasi = (TP + TN) / Total Data Testing
         = (74 + 167) / 257
         = 241 / 257
         = 93,77%

b. Presisi

Presisi = TP / (TP + FP)
         = 74 / (74 + 16)
         = 74 / 90
         = 82,22%

c. Recall

Recall = TP / (TP + FN)
        = 74 / (74 + 0)
        = 74 / 74
        = 100,00%

d. F1-Score

F1-Score = (2 × Presisi × Recall) / (Presisi + Recall)
          = (2 × 82,22 × 100) / (82,22 + 100)
          = 16.444 / 182,22
          = 90,24%
Tabel 4.16 – Rekapitulasi Metrik Evaluasi Model Naive Bayes

Sumber: Hasil Pengolahan Data (2025)

Berdasarkan Tabel 4.16, model Naive Bayes menghasilkan akurasi sebesar 93,77%, yang berarti dari 257 data uji sebanyak 241 data berhasil diklasifikasikan dengan benar. Nilai Recall sebesar 100% menunjukkan bahwa seluruh calon mahasiswa yang sesungguhnya MASUK berhasil diidentifikasi model tanpa ada yang terlewat (FN = 0), sehingga tidak ada calon mahasiswa potensial yang luput dari rekomendasi tindak lanjut.

Nilai Presisi sebesar 82,22% menunjukkan bahwa 82,22% dari seluruh prediksi MASUK memang benar-benar terbukti MASUK. Sementara nilai F1-Score sebesar 90,24% mencerminkan keseimbangan yang baik antara presisi dan recall. Secara keseluruhan, hasil evaluasi ini menunjukkan bahwa algoritma Naive Bayes mampu mengidentifikasi pola kecenderungan konfirmasi pendaftaran mahasiswa baru secara efektif dan layak digunakan sebagai dasar sistem rekomendasi tindak lanjut.
