C. Hasil Analisis Metode

1. Persiapan Data

Dataset yang digunakan dalam penelitian ini bersumber dari data historis pendaftaran mahasiswa baru Universitas Binaniaga Indonesia. Total keseluruhan data berjumlah 1.285 record dengan lima atribut fitur dan satu atribut target (label kelas). Atribut yang digunakan sebagai fitur meliputi: kategori jarak asal, tingkat follow up internal, status test, kategori nilai test, dan kategori penghasilan orang tua. Atribut target adalah status retensi final yang terdiri dari dua kelas, yaitu MASUK dan TIDAK MASUK.

Pembagian dataset dilakukan dengan rasio 80:20, yaitu 80% data digunakan sebagai data training dan 20% digunakan sebagai data testing. Berdasarkan rasio tersebut diperoleh data training sebanyak 1.028 record dan data testing sebanyak 257 record.

3. Perhitungan Probabilitas Prior

Probabilitas prior dihitung berdasarkan proporsi kemunculan masing-masing kelas pada data training yang berjumlah 1.028 record, dengan rincian 227 record berkelas MASUK dan 801 record berkelas TIDAK MASUK. Perhitungan probabilitas prior menggunakan rumus sebagai berikut.

P(Ck) = nk / N

Keterangan: nk = jumlah data kelas Ck pada data training; N = total data training.

Berdasarkan rumus tersebut, diperoleh nilai probabilitas prior untuk masing-masing kelas sebagai berikut.

P(MASUK) = 227 / 1.028 = 0,2208
P(TIDAK MASUK) = 801 / 1.028 = 0,7792

4. Perhitungan Probabilitas Kondisional

Probabilitas kondisional dihitung menggunakan Laplace Smoothing guna menghindari nilai probabilitas nol yang dapat terjadi apabila suatu nilai atribut tidak muncul pada kelas tertentu di data training. Rumus yang digunakan adalah sebagai berikut.

P(xi | Ck) = (nik + 1) / (nk + K)

Keterangan: nik = jumlah kemunculan nilai xi pada kelas Ck; nk = total data kelas Ck; K = jumlah nilai unik pada fitur tersebut.

5. Contoh Perhitungan Manual Klasifikasi

Sebagai ilustrasi penerapan algoritma Naive Bayes, berikut disajikan contoh perhitungan klasifikasi untuk satu data calon mahasiswa dengan karakteristik sebagai berikut.

Tabel 4.13 – Data Calon Mahasiswa untuk Contoh Klasifikasi
AtributNilai
Kategori Jarak AsalDekat
Tingkat Follow Up InternalBelum Dihubungi
Status TestSudah Tes
Kategori Nilai TestNilai Tinggi
Kategori Penghasilan Orang TuaTidak Diketahui
Sumber: Data Internal Pendaftaran Universitas Binaniaga Indonesia

Langkah 1 – Menghitung Probabilitas Prior

Probabilitas prior masing-masing kelas dihitung dari data training (1.028 record).

P(MASUK) = 227 / 1.028 = 0,2208
P(TIDAK MASUK) = 801 / 1.028 = 0,7792

Langkah 2 – Menghitung Probabilitas Kondisional untuk Kelas MASUK

Setiap probabilitas kondisional dihitung menggunakan Laplace Smoothing: P(xi | Ck) = (nik + 1) / (nk + K).

P(Jarak = Dekat  | MASUK)         = (122 + 1) / (227 + 3) = 123 / 230 = 0,5348
P(Follow Up = Belum Dihubungi | MASUK) = (196 + 1) / (227 + 4) = 197 / 231 = 0,8528
P(Status Test = Sudah Tes     | MASUK)         = (227 + 1) / (227 + 2) = 228 / 229 = 0,9956
P(Nilai = Nilai Tinggi       | MASUK)         = (216 + 1) / (227 + 3) = 217 / 230 = 0,9435
P(Penghasilan = Tidak Diketahui | MASUK) =  (64 + 1) / (227 + 6) =  65 / 233 = 0,2790

Nilai skor MASUK diperoleh dari perkalian probabilitas prior dengan seluruh probabilitas kondisional:

P(MASUK | X) = P(MASUK) × P(Dekat|M) × P(Belum Dihubungi|M) × P(Sudah Tes|M) × P(Nilai Tinggi|M) × P(Tidak Diketahui|M)
= 0,2208 × 0,5348 × 0,8528 × 0,9956 × 0,9435 × 0,2790
= 0,026415

Langkah 3 – Menghitung Probabilitas Kondisional untuk Kelas TIDAK MASUK

P(Jarak = Dekat  | TIDAK MASUK)         = (329 + 1) / (801 + 3) = 330 / 804 = 0,4104
P(Follow Up = Belum Dihubungi | TIDAK MASUK) = (701 + 1) / (801 + 4) = 702 / 805 = 0,8720
P(Status Test = Sudah Tes     | TIDAK MASUK)         =  (78 + 1) / (801 + 2) =  79 / 803 = 0,0984
P(Nilai = Nilai Tinggi       | TIDAK MASUK)         =  (67 + 1) / (801 + 3) =  68 / 804 = 0,0846
P(Penghasilan = Tidak Diketahui | TIDAK MASUK) = (791 + 1) / (801 + 6) = 792 / 807 = 0,9814

Nilai skor TIDAK MASUK diperoleh dari:

P(TM | X) = P(TM) × P(Dekat|TM) × P(Belum Dihubungi|TM) × P(Sudah Tes|TM) × P(Nilai Tinggi|TM) × P(Tidak Diketahui|TM)
= 0,7792 × 0,4104 × 0,8720 × 0,0984 × 0,0846 × 0,9814
= 0,002277

Langkah 4 – Penentuan Kelas

Kelas dipilih berdasarkan nilai skor yang paling tinggi di antara kedua kelas.

P(MASUK | X)  = 0,026415
P(TM | X)     = 0,002277
0,026415 > 0,002277 → Kelas Prediksi: MASUK

Kesimpulan: Karena nilai P(MASUK) lebih besar dari P(TIDAK MASUK), maka calon mahasiswa dengan karakteristik tersebut diklasifikasikan ke dalam kelas MASUK.

6. Evaluasi Model

Model Naive Bayes yang telah dibangun selanjutnya dievaluasi menggunakan data testing sebanyak 257 record. Evaluasi dilakukan menggunakan confusion matrix untuk mengukur performa model secara menyeluruh.

Tabel 4.14 – Confusion Matrix Hasil Klasifikasi Naive Bayes
Aktual Prediksi
MASUK TIDAK MASUK
MASUK 74 (TP) 0 (FN)
TIDAK MASUK 16 (FP) 167 (TN)
Sumber: Hasil Pengolahan Data (2025)

Berdasarkan confusion matrix pada Tabel 4.14, diperoleh nilai True Positive (TP) sebesar 74, True Negative (TN) sebesar 167, False Positive (FP) sebesar 16, dan False Negative (FN) sebesar 0. Berdasarkan nilai-nilai tersebut, dihitung metrik evaluasi sebagai berikut.

a. Akurasi

Akurasi = (TP + TN) / Total = (74 + 167) / 257 = 241 / 257 = 93,77%

b. Presisi

Presisi = TP / (TP + FP) = 74 / (74 + 16) = 74 / 90 = 82,22%

c. Recall

Recall = TP / (TP + FN) = 74 / (74 + 0) = 74 / 74 = 100,00%

d. F1-Score

F1-Score = (2 × Presisi × Recall) / (Presisi + Recall) = (2 × 82,22 × 100) / (82,22 + 100) = 90,24%

Berdasarkan hasil evaluasi pada Tabel 4.15, model Naive Bayes menghasilkan akurasi sebesar 93,77%, yang berarti dari 257 data uji, sebanyak 241 data berhasil diklasifikasikan dengan benar. Nilai Recall sebesar 100% menunjukkan bahwa seluruh calon mahasiswa yang sesungguhnya akan MASUK berhasil diidentifikasi oleh model tanpa ada yang terlewat (nilai FN = 0). Hal ini sangat penting dalam konteks rekomendasi tindak lanjut, karena memastikan tidak ada calon mahasiswa potensial yang tidak terdeteksi.

Nilai Presisi sebesar 82,22% menunjukkan bahwa dari seluruh calon mahasiswa yang diprediksi MASUK, sebesar 82,22% di antaranya memang benar-benar melakukan konfirmasi pendaftaran. Sementara itu, nilai F1-Score sebesar 90,24% mencerminkan keseimbangan yang baik antara presisi dan recall, yang mengindikasikan bahwa model mampu bekerja secara efektif pada kedua aspek tersebut.

Secara keseluruhan, hasil evaluasi ini menunjukkan bahwa algoritma Naive Bayes mampu mengidentifikasi pola kecenderungan konfirmasi pendaftaran mahasiswa baru secara efektif berdasarkan atribut-atribut yang tersedia dalam data historis. Dengan tingkat akurasi yang tinggi dan recall yang sempurna, model ini layak digunakan sebagai dasar dalam membangun sistem rekomendasi tindak lanjut konfirmasi pendaftaran.
